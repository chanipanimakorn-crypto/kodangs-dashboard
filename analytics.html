KODANGS ANALYTICS FRAMEWORK

Data ที่ต้องมี เพื่อ scale ให้ถึง 4-6M

สร้างเมื่อ 14 เมษายน 2569 · inspired by ดอกสตรีม dashboard model
1 Product Profitability · สินค้าไหนทำกำไร PRIORITY 1
เทียบกับร้านอาหาร: "เมนูไหนทำเงิน เมนูไหนควรตัดออก"
ยอดขายต่อ SKUชิ้น + บาท แยกสี/ไซส์Shipnity
ต้นทุนต่อ SKUผ้า + เย็บ + พิมพ์ + แพ็คManual
กำไรขั้นต้น / SKUบาท + % marginคำนวณ
Sell-through Rateผลิตเท่าไหร่ ขายได้กี่ %Shipnity
Dead Stock Flagค้าง > 90 วัน / 180 วันShipnity
Revenue per Styleรวมทุกสี ทุกไซส์ ต่อ designShipnity
ตัวไหนควรผลิตเพิ่ม ตัวไหนควรหยุด ตัวไหนควรลด markdown?
2 Inventory Health · สต็อกครบไหม PRIORITY 2
เทียบกับร้านอาหาร: "ครอสเช็คกับครัวว่าซื้อของมาครบไหม"
สต็อกคงเหลือแยก SKU / สี / ไซส์Shipnity
Days of Inventoryสต็อกที่มี ขายได้อีกกี่วันคำนวณ
Reorder Alertตัวไหนใกล้หมด สั่งผลิตด่วนคำนวณ
Size Curveไซส์ไหนหมดก่อน ไหนค้างShipnity
Stock ReconciliationShipnity vs. ของจริงตรงไหมManual
Aging Reportสต็อกอายุ 30/60/90/180 วันShipnity
S หมดทุกตัว สั่งผลิตเพิ่มได้เลย? XL เหลือเยอะ ต้องดันขาย?
3 Channel Performance · ช่องทางไหนคุ้ม PRIORITY 3
เทียบกับร้านอาหาร: "ขาย dine-in vs. delivery vs. GrabFood อันไหนกำไรจริง"
ยอดขายแยก ChannelShopee / Lazada / TikTok / LINE@ / Website / หน้าร้านShipnity
กำไรสุทธิต่อ Channelหัก GP, ค่าส่ง, ค่าโปรคำนวณ
CAC per Channelต้นทุนได้ลูกค้าใหม่Manual
Return Rateช่องไหนคืนสินค้าเยอะShipnity
Shopee ยอดเยอะ แต่หัก GP แล้วกำไรจริงน้อยกว่า LINE@?
4 Cost Breakdown · วัตถุดิบอะไรแพง PRIORITY 4
เทียบกับร้านอาหาร: "วัตถุดิบชิ้นไหนแพงเป็นพิเศษ"
ต้นทุนผ้า / เมตรแยกตาม supplierManual
ต้นทุนเย็บ / ตัวแยกตามโรงงานManual
Accessories Costซิป, กระดุม, label, hang tagManual
Cost Trend M/Mเปรียบเทียบเดือนต่อเดือนคำนวณ
ทำไมต้นทุนเดือนนี้สูงกว่าเดือนก่อน? ผ้าล็อตไหนขึ้นราคา?
5 Collection Performance · คอลฯ ไหนเวิร์ค PRIORITY 5
เทียบกับร้านอาหาร: "เซตเมนูไหนขายดี ควร rotate ยังไง"
ยอดขายต่อ Collectionแยกตาม drop / seasonShipnity
First-Week Sell-throughสัปดาห์แรกขายได้กี่ %Shipnity
Repeat Purchaseลูกค้าซื้อซ้ำจากคอลฯ เดิมShipnity
Best & Worst per Droptop 3 / bottom 3 SKUคำนวณ
Camo collection ขายดีกว่า basic 3 เท่า — ควร drop ต่อเนื่อง?
6 Production & Supplier · โรงงานส่งตรงเวลาไหม PRIORITY 6
เทียบกับร้านอาหาร: "supplier ส่งวัตถุดิบตรงเวลาไหม คุณภาพโอเคไหม"
Lead Time จริง vs. สัญญาสั่งวันไหน ได้ของวันไหนManual
Defect Rateของเสียกี่ % ต่อ supplierManual
Cost Comparisonเทียบราคาระหว่าง supplierManual
โรงงาน A ส่งช้าแต่ของดี vs. โรงงาน B เร็วแต่ defect เยอะ — เลือกใคร?
สรุป Priority
#หมวดทำไมสำคัญแหล่งข้อมูล
1Product Profitabilityรู้ว่าตัวไหนทำเงิน ตัวไหนตัด — ตัดสินใจผลิตได้เลยShipnity + Manual
2Inventory Healthสต็อกหมด = เสียยอด / สต็อกค้าง = จมทุนShipnity
3Channel Performanceรู้ว่าควรเทแรงช่องไหนShipnity
4Cost Breakdownควบคุม margin ละเอียดขึ้นManual
5Collection Performanceวางแผน drop ถัดไปแม่นขึ้นShipnity
6Production Trackingscale ไปต้องมี แต่ยังไม่เร่งManual
Next Steps · สิ่งที่ต้องทำต่อ
1ดึง Export ข้อมูลจาก Shipnity
export รายงานสินค้า (ยอดขาย + สต็อก) ออกมาเป็น CSV/Excel เพื่อให้ Claude วิเคราะห์ SKU profitability + inventory health ได้ทันที
สัปดาห์นี้
2จัดทำ Cost Sheet ต่อ SKU
รวบรวมต้นทุนจริงต่อตัว (ผ้า + เย็บ + accessories + แพ็ค) ใส่ Google Sheet — อันนี้ Shipnity ไม่มี ต้อง input เอง แต่ทำครั้งเดียวใช้ได้ตลอด
สัปดาห์นี้ – สัปดาห์หน้า
3สร้าง Live Dashboard รอบแรก
เอาข้อมูลจากข้อ 1 + 2 มาสร้าง dashboard จริง — เริ่มจาก Product Profitability + Inventory Health ก่อน เพราะ impact สูงสุด
สัปดาห์หน้า
4เพิ่ม Channel Performance
ดึงยอดขายแยก channel จาก Shipnity + ใส่ค่า GP/commission ต่อ platform เพื่อเห็นกำไรจริงต่อช่อง
ภายในเดือน เม.ย.
5ผูก Auto-Update เข้า Morning Brief
พอ dashboard เสถียร ให้ดึงข้อมูล key metrics ใส่ morning brief อัตโนมัติ — เห็นทุกเช้าโดยไม่ต้องเปิดหลายหน้า
